## 핵심 처리 로직: 동시복용 약물 수 산출

```
입력: 환자별 처방 이력 (약품코드, 처방시작일, 투약일수)

1. 각 처방에 대해 활성 기간(active period) 생성
   active_start = 처방시작일 (진료개시일)
   active_end   = 처방시작일 + 총투약일수

2. 기준일(index_date) 설정
   - 분석 기준일 또는 슬라이딩 윈도우

3. 기준일에 활성화된 약물 목록 산출
   WHERE active_start <= index_date AND active_end > index_date

4. 고유 성분 수 집계 (ATC 5단계 기준 DISTINCT COUNT)

5. 기준기간 내 최대/평균 동시복용 약물 수 산출
   - 매일(daily) 활성 약물 수를 계산하여 MAX/AVG 집계
```

## 기술 스택 요약

| 구성요소 | 기술 | 비고 |
|----------|------|------|
| ETL/처리 | Apache Spark 3.x | PySpark |
| 저장소 | HDFS / S3 + Parquet | Delta Lake 고려 |
| 피처 스토어 | Custom (Parquet) 또는 Feast | 규모에 따라 결정 |
| 스케줄링 | Apache Airflow | DAG 기반 파이프라인 관리 |
| 품질 관리 | Great Expectations | 데이터 검증 자동화 |
| 모니터링 | Grafana + Prometheus | 파이프라인 헬스 모니터링 |
| 보안 | Apache Ranger / RBAC | 접근 통제 |
